#+title: .emacs.d

Personal Emacs configuration — literate, tangled from org-mode.

The entire configuration lives in [[file:config.org][config.org]]. =init.el= is a thin loader that
tangles and loads it via =org-babel-load-file=.

* Layout

- [[file:init.el][=init.el=]] — loader: redirects =custom-file=, tangles and loads =config.org=
- [[file:config.org][=config.org=]] — the actual configuration (tangles to =config.el= on load)
- [[file:early-init.el][=early-init.el=]] — frame/UI setup before package load
- [[file:flake.nix][=flake.nix=]] — builds Emacs + ELisp packages + the tool closure; exposes a home-manager module
- [[file:nix/hm-module.nix][=nix/hm-module.nix=]] — home-manager module (see [[*Nix / home-manager][Nix / home-manager]])

* First run

Emacs packages are provided by Nix (not [[https://github.com/radian-software/straight.el][straight.el]]) — the flake builds Emacs
with every =use-package= dependency on =load-path=. On startup =emacs= only:

1. Tangles =config.org= → =config.el=
2. Loads it; =use-package= forms =require= packages already provided by Nix
   (they carry no =:ensure=/=:straight= and install nothing)

To add a package, add it to the =withPackages= list in [[file:flake.nix][=flake.nix=]] and rebuild.

* Highlights

- Evil-mode with =SPC= leader
- Vertico / Orderless / Corfu / Consult / Embark / Marginalia completion stack
- Catppuccin theme, auto-switches Latte/Mocha with system appearance (via =auto-dark=, cross-platform)
- Eglot for C#, Nix, and Markdown (marksman); lsp-mode for TypeScript/TSX (started manually via =M-x eglot= / =M-x lsp=, no auto-start)
- Tree-sitter via =treesit-auto=
- Magit, diff-hl, Apheleia formatting on save

* Nix / home-manager

The flake owns Emacs, the ELisp packages, the *external tool closure* (language
servers), and the placement of this config — all exposed as a home-manager
module.

** Emacs and packages

=packages.<system>.emacs= (and =overlays.default='s =emacs-dotemacs=) is one
Emacs for Linux and Darwin, built with every =use-package= dependency on
=load-path= (see the =withPackages= list in [[file:flake.nix][=flake.nix=]]). There is no straight.el:
=use-package= forms carry no =:ensure=/=:straight= and install nothing. Adding a
package means adding it to that list and rebuilding. =lsp-tailwindcss= (not in
nixpkgs) is built from source via =fetchFromGitHub=; =vterm='s native module is
compiled by Nix rather than at runtime.

On Darwin the NS build already provides =ns-appearance= / =ns-transparent-titlebar=;
the overlay adds the emacs-plus =system-appearance= patch on top so
=ns-system-appearance= exists too, giving =auto-dark= an event-driven light/dark
source. The patch touches Cocoa, so it is applied on Darwin only — Linux builds a
normal PGTK Emacs from the same derivation.

Using the Nix Emacs is opt-in per host: leave =programs.dotemacs.package = null=
(default) to keep an externally-managed Emacs, or set it to =pkgs.emacs-dotemacs=
(with =overlays.default= applied) to have Nix provide Emacs and all packages.

** Wiring into home-manager

#+begin_src nix
# flake inputs
dotemacs.url = "github:hadyny/.emacs.d";
dotemacs.inputs.nixpkgs.follows = "nixpkgs";

# apply the overlay so `pkgs.emacs-dotemacs` (Emacs + all packages) is available
nixpkgs.overlays = [ inputs.dotemacs.overlays.default ];

# home-manager modules
imports = [ inputs.dotemacs.homeModules.default ];

programs.dotemacs = {
  enable = true;
  # Absolute path to this checkout, linked as ~/.emacs.d via an out-of-store
  # (writable) symlink so config.org edits take effect without a rebuild and
  # the tangled config.el can be written next to it.
  configPath = "/Users/you/src/dotemacs.d";
  # Set to [ ] when a shared package list already installs the language servers.
  # tools = [ ];
  # Nix-provided Emacs + all packages (else leave null for an external Emacs):
  package = pkgs.emacs-dotemacs;
};
#+end_src

Module options (=programs.dotemacs.*=):

- =enable= — install the tool closure and manage =~/.emacs.d=
- =tools= — external tools / language servers on =PATH= (default: marksman,
  roslyn-ls, tailwindcss-language-server, vscode-langservers-extracted,
  coreutils-prefixed). Set to =[ ]= to defer to another package list.
- =configPath= — absolute path to a live checkout; =null= links the flake's
  read-only source instead (which can't be tangled into — see the option doc)
- =package= — Emacs package to install via nix; =null= (default) keeps an
  externally-managed Emacs. Set to =pkgs.emacs-dotemacs= (with the overlay) for
  the cross-platform build with all packages.

Other flake outputs:

- =nix flake check= — runs =checks.smoke= (tangle + byte-compile),
  =checks.appearance= (theme switch stays cross-platform), and
  =checks.packages-loadable= (the Nix Emacs has every config package, incl.
  lsp-tailwindcss and vterm, on =load-path=)
- =nix develop= — dev shell with headless Emacs + the language servers
- =nix fmt= — format the Nix files (=nixfmt=)
- =packages.<system>.emacs= — the cross-platform Emacs with all packages
- =packages.<system>.emacs-bare= — Emacs with only the Darwin appearance patch
- =packages.<system>.emacs-tools= — the tool closure as one joined derivation
- =overlays.default= — overrides =pkgs.emacs= with the patched build
